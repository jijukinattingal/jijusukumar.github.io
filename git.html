5 Show commit logs

git log show recent commits, most recent on top. Useful options: –color with color –graph with an ASCII-art commit graph on the left –grep grep –decorate with branch and tag names on appropriate commits –stat with stats (files changed, insertions, and deletions) -p with full diffs –author=foo only by a certain author –after="MMM DD YYYY" ex. ("Jun 20 2008") only commits after a certain date –before="MMM DD YYYY" only commits that occur before a certain date –merge only the commits involved in the current merge conflicts

5.1 Examples:

5.1.1 Show branch/tag info

git log --decorate

5.1.2 Grep a string in the git commit message

git log --grep="DSDS"

5.1.3 Graph

git log --graph --color --decorate --abbrev-commit telephony/java/com/android/internal/telephony/gsm/GsmDataConnectionTracker.java

git log --graph --color --decorate --pretty=oneline telephony/java/com/android/internal/telephony/gsm/GsmDataConnectionTracker.java

git log --graph --color --decorate --oneline telephony/java/com/android/internal/telephony/gsm/GsmDataConnectionTracker.java

git log --branches --graph --color --decorate --oneline telephony/java/com/android/internal/telephony/gsm/GsmDataConnectionTracker.java

5.1.4 List of change between the AUS; export to xls sheet

git log --no-merges AU_LINUX_ANDROID_JB_2.5.04.02.02.040.400..AU_LINUX_ANDROID_JB_2.5.04.02.02.040.443 --date=short --pretty=format:"%H%x09%an%x09%ad%x09%s" > commitLog.xls

6 Show the files modified:

git show --pretty="format:" --name-only <sha>

6.1 Examples:

6.1.1 List all the files changed in a commit

git show --pretty="format:" --name-only 4b29d6554c26c5537425159a1f61c1c8e03d9045

6.1.2 List all the new files add in a commit

git show --pretty="format:" --name-only --diff-filter=A 4b29d6554c26c5537425159a1f61c1c8e03d9045

6.1.3 List all the modified files add in a commit

git show --pretty="format:" --name-only --diff-filter=M 4b29d6554c26c5537425159a1f61c1c8e03d9045

9 Undo all changes in A FILE

10 Fixing un-committed mistakes

git reset --hard HEAD

This will undo all the changes and return the entire working tree to the last committed state

11 Apply the patch

patch   -p1   <  filename.patch 

where 
filename.patch is a patch file generated by
        git diff > filename.patch  or
        git show <sha id> > filename.patch

14 How to upload a change

  1. Create a branch

    repo start <branch-name> .

  2. Modifiy the files
  3. Commit the changes

    git commit -a

    Give the below details:

    module: Brief desciption of changes

    Description

    CRs-Fixed: CR#

  4. repo upload .

To upload a patch (code change after review comments) in the gerrit

  1. git commit –amend -a Optional: change the description.
  2. repo upload .

23 Rebase

Instructions to rebase:

"Create a new directory and 'cd' to it." 

git init
git fetch ssh://review-android.quicinc.com:29418/platform/vendor/qcom-proprietary/ship/qcril refs/heads/kk_2.7
git checkout FETCH_HEAD
git fetch ssh://review-android.quicinc.com:29418/platform/vendor/qcom-proprietary/ship/qcril refs/changes/34/623534/1
git cherry-pick FETCH_HEAD
"Modify files that have conflicts."
git add .
git commit -c FETCH_HEAD
git push ssh://review-android.quicinc.com:29418/platform/vendor/qcom-proprietary/ship/qcril HEAD:refs/changes/623534

Replace "ssh://visweswa@review-android.quicinc.com:29418" with "git://git.quicinc.com", this will get your changes pulled faster and no need to rely on Server. (which inturn reduces load on server)
