1 Search and Find

1.1 Search a pattern

In <ESC> mode /pattern

1.2 Search and replace a pattern

Current line:s/pattern/replace/
With prompt:s/pattern/replace/c
Multiple occurrence in current line:s/pattern/replace/g
Whole file:1,$s/pattern/replace/g
:%s/pattern/replace/g
:%s/pattern/replace/cg

1.3 Search and Replace with the 'matched string + new string'

:%s/<PATTERN>/&<NEW_STRING>/ The & represents the entire text which was matched.

Eg: 1,$s/^[0-9]\+\t/|-\r|& /

1.4 Finding a whole word

\< represents the beginning of a word, and \> represents the end of a word

so to find only the whole word "word" - use the pattern: \<word\>

1.5 Search multiple patterns:

vimgrep: to search multiple patterns:

How to use :vimgrep to get a list of files containing a certain pattern. :vimgrep 'pattern' % The results of a :vimgrep search are stored in the "quickfix" list. This list can be displayed in a second Vim window by :copen

Eg: :vimgrep 'pattern' %

To open the search results :copen If you want to strip the file name, line and col number from the search result :set modifiable :1,$s/^.* col [0-9]\+| //

1.6 Search and Delete

Delete every line in the file that match a pattern

:g/pattern/d

Delete every line in the file that does not match a pattern

:v/patter/d :g!/pattern/d

3 Highlight multiple words

:highlight MyColorGroup ctermbg=green guibg=green :highlight MyColorGroup2 ctermbg=red guibg=red :highlight MyColorGroup3 ctermbg=gray guibg=gray

:match MyColorGroup SETUP_DATA_CALL :2match MyColorGroup2 DEACTIVATE_DATA_CALL :3match MyColorGroup3 UNSOL_DATA_CALL

4 Using marks

To jump to a mark enter an apostrophe (') or backtick (`) followed by a letter. Using an apostrophe jumps to the beginning of the line holding the mark, while a backtick jumps to the line and column of the mark. Using a lowercase letter (for example `a) will only work if that mark exists in the current buffer. Using an uppercase letter (for example `A) will jump to the file and the position holding the mark (you do not need to open the file prior to jumping to the mark).

  • Each file can have mark a – use a lowercase mark to jump within a file.
  • There is only one file mark A – use an uppercase mark to jump between files.
CommandDescription
maset mark a at current cursor location
'ajump to line of mark a (first non-blank character in line)
`ajump to position (line and column) of mark a
d'adelete from current line to line of mark a
d`adelete from current cursor position to position of mark a
c'achange text from current line to line of mark a
y'ayank text to unnamed buffer from cursor to position of mark a
:markslist all the current marks
:marks aBlist marks a, B

Ref: http://vim.wikia.com/wiki/Using\_marks

6 File format

add the 

set ffs=unix  set fileformat=unix

in ~/.vimrc

then open the file, it will show you ^M at the end of the line if it is in dos format.

7 Miscellaneous

7.1 How to Convert the content of the file to HTML:

Many file types can be displayed with syntax highlighting. :help syntax Vim provides a script that can create an html document (including the foreground and background colors, and syntax highlighting) from the current file, or from selected lines. :help convert-to-HTML In gvim, use "Convert to HTML" on the Syntax menu. A new file is created, containing the html equivalent of all text from the current buffer. The same operation can be performed by entering a command: :TOhtml

7.2 Add line number
